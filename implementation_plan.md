# แผนพัฒนาระบบ อุปกรณ์เสริม, การตั้งค่าแบบ Dynamic, และระบบคลังเก็บของรวม

เป้าหมายคือการเพิ่มความซับซ้อนให้ระบบฟาร์มของ (Idle) โดยเพิ่มเงื่อนไขของอุปกรณ์สวมใส่ การตั้งค่าตัวแปรของเซิร์ฟเวอร์แบบ Real-time ผ่าน MongoDB และระบบกระเป๋าที่มีข้อจำกัด

## User Review Required

> [!WARNING]
> **กรุณาตรวจสอบโครงสร้างที่ออกแบบไว้ หากมีจุดใดไม่ตรงกับความต้องการ โปรดแจ้งให้ปรับแก้ก่อนเริ่มเขียนโค้ด:**
> 1. **เงื่อนไขช่องเก็บของ (5 ช่อง):** โดยปกติ 1 ช่อง = 1 ชนิดไอเทม (Stack ทับกันได้) ถ้าเริ่มเกมแจกอุปกรณ์ 3 อย่าง (ขวาน, ที่ขุด, ดาบ) ผู้เล่นจะเหลือที่ว่างเพียง 2 ช่องสำหรับเก็บทรัพยากร (เช่น ไม้, หิน) รูปแบบนี้ตรงกับที่คิดไว้หรือไม่? (ถ้าใช่ หมายความว่าผู้เล่นต้องรีบฝากของเข้าคลังรวม หรืออัปเกรดกระเป๋า)
> 2. **เป้าหมายคลังรวม (Global Storage):** คลังรวมนี้จะเป็น "กองกลางของเซิร์ฟเวอร์" ที่ผู้เล่น *ทุกคน* ฝากและถอนได้อิสระใช่หรือไม่? (เช่น A ฝากไม้ไว้ B สามารถมาหยิบไม้นั้นไปใช้ได้เลย)
> 3. **อุปกรณ์ช่วยคูณทรัพยากร:** สูตรที่เตรียมไว้คือ `จำนวนทรัพยากร = (เวลาที่ผ่านไป / เวลาพื้นฐาน) * เลเวลอุปกรณ์` เช่น ถ้าขวานเลเวล 2 ก็จะได้ไม้ไวขึ้น 2 เท่าจากปกติ โอเคไหมครับ?

## Proposed Changes

### Database Schema (MongoDB Updates)

#### [NEW] Collection `configs`
บันทึกตัวแปรทุกอย่างของเซิร์ฟเวอร์ใน Document เดียว (เพื่อให้แก้ข้อมูลผ่าน Database Tool อย่าง MongoDB Compass ได้ทันทีแล้วเซิร์ฟเวอร์ใช้ค่าใหม่เลยโดยไม่ต้อง Restart โค้ด)
- **`actions`**: เก็บตั้งค่าเวลาดรอปของ เช่น `cutting_wood` ใช้ `wooden_axe` และดรอป `wood` ทีละ 1 ชิ้น ทุกๆ 10 วินาที
- **`upgrades`**: บอกว่าอุปกรณ์แต่ละชนิด ใช้ทรัพยากรในการอัปเกรดเท่าไหร่
- **`inventory`**: กำหนด `base_slots` = 5

#### [NEW] Collection `global_storage`
- บันทึกไอเทมกองกลางของเซิร์ฟเวอร์
- ฟิลด์: `item_id`, `quantity` (รวมของทุกคนไว้ในที่เดียวกัน)

#### [MODIFY] Collection `characters`
- เพิ่มฟิลด์ `max_inventory_slots` (เริ่มต้นที่ 5) เข้าไปในข้อมูลตัวละคร

---

### Backend Components (Go)

#### [NEW] `models/config.go`
เพิ่ม Struct สำหรับรองรับข้อมูลจาก Collection `configs` 

#### [MODIFY] `handlers/auth.go`
- **Signup API**: ปรับรายการของเริ่มต้น ให้แจก `wooden_axe` (ขวานไม้), `wooden_pickaxe` (ที่ขุดไม้), `wooden_sword` (ดาบไม้) พร้อมตั้ง `max_inventory_slots = 5`

#### [MODIFY] `handlers/character.go`
- **Claim API**: 
  - เชื่อมโยงเข้ากับ Collection `configs` เพื่อดูว่าผู้เล่นทำ Action อะไรอยู่ ต้องใช้อุปกรณ์ชิ้นไหน
  - ดึง "เลเวลอุปกรณ์" จากกระเป๋ามาคูณความเร็วในการฟาร์ม
  - **จำกัดช่อง**: ตรวจสอบว่าถ้ากระเป๋าเต็ม (ชนิดไอเทมเกิน `max_inventory_slots`) และเป็นของชนิดใหม่ จะไม่สามารถ Claim ได้ ระบบจะฟ้องว่ากระเป๋าเต็ม

#### [NEW] `handlers/storage.go`
สร้าง Endpoint สำหรับระบบคลังกองกลาง
- **`GET /api/storage`** : ดูของในคลังรวม
- **`POST /api/storage/deposit`** : ฝากของเข้าคลังรวม (หักจากกระเป๋าตัวเอง เพิ่มเข้ากองกลาง)
- **`POST /api/storage/withdraw`** : ถอนของจากคลังรวมเข้ากระเป๋า (หักจากกองกลาง เพิ่มเข้าตัวเอง)

#### [MODIFY] `routes/routes.go`
- เพิ่ม Route สำหรับระบบ `storage`

## Verification Plan

1. **ระบบเริ่มเกม**:
   - ทดสอบสร้างตัวละครใหม่ -> เช็กว่ามีขวาน, ที่ขุด, ดาบ และมีช่องกระเป๋าจำกัดที่ 5
2. **ระบบดรอปและคูณอัตรา (Multiplier)**:
   - ทดสอบแก้ไขเวลาดรอปใน DB แบบ Real-time แล้วกด Claim -> ได้ของตามที่แก้ 
   - อัปเกรดขวานเป็น Lv.2 แล้วปล่อยตัดไม้ -> ได้ไม้สองเท่าเมื่อเทียบกับขวาน Lv.1 ในเวลาเท่ากัน
3. **ระบบกระเป๋าเต็ม และคลังรวม**:
   - ฟาร์มให้กระเป๋าเต็ม (ครบ 5 ชนิด) -> ระบบต้องไม่ยอมให้รับทรัพยากรชนิดที่ 6
   - นำของไปฝาก `global_storage` -> ของในตัวหายไป -> กระเป๋ามีที่ว่าง -> กลับไป Claim ใหม่ได้
   - ให้อีกไอดีนึงมาลองหยิบของออกจาก `global_storage` -> สามารถหยิบได้สำเร็จ
